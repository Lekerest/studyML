**Дата:** 03_03_2026

## 🎯 Цель работы

Решить задачу бинарной классификации на датасете **Breast Cancer Wisconsin** и подобрать такую стратегию, которая **минимизирует FN (false negative) для злокачественной опухоли**, то есть **максимизирует Recall для malignant**.

Это означает, что основная цель работы — **сократить число случаев, когда злокачественная опухоль ошибочно предсказывается как доброкачественная**.

В рамках лабораторной работы необходимо:

- выбрать подходящую стратегию классификации  
- проанализировать признаки и попробовать сократить их число  
- сравнить модель на всех признаках и на отобранных признаках  
- подобрать порог классификации `threshold`, чтобы уменьшить FN  
- оценить итоговую модель по метрикам и confusion matrix  
- сделать вывод, какая стратегия лучше соответствует ТЗ  

---

## 📚 Датасет

Использован датасет **Breast Cancer Wisconsin** (`load_breast_cancer` из `sklearn.datasets`).

В данных:
- **569 объектов**
- **30 числовых признаков**
- **2 класса**

Исходная кодировка классов:
- `0 = malignant` — злокачественная опухоль  
- `1 = benign` — доброкачественная опухоль  

Для удобства решения задачи целевая переменная была переопределена так:

- `1 = malignant`
- `0 = benign`

Это было сделано для того, чтобы метрика **Recall** считалась именно для злокачественных опухолей как для положительного класса.

---

## 🛠 Выбранная стратегия

Для решения задачи была выбрана **логистическая регрессия с L2-регуляризацией**.

### Почему выбрана именно эта стратегия

- логистическая регрессия хорошо подходит для бинарной классификации  
- модель интерпретируема: можно анализировать коэффициенты признаков  
- L2-регуляризация делает модель более устойчивой и не зануляет признаки слишком агрессивно  
- параметр `class_weight='balanced'` помогает модели внимательнее относиться к классу `malignant`  
- использование `predict_proba()` и подбор `threshold` позволяет напрямую управлять компромиссом между Recall и Accuracy

## 📌 Интерпретация

### Что означает `threshold`
Порог `threshold` определяет, начиная с какой вероятности модель относит объект к классу `malignant`.

- если вероятность `>= threshold` → `malignant`
- если вероятность `< threshold` → `benign`

### Почему уменьшение threshold помогает
При понижении threshold модель чаще предсказывает `malignant`, поэтому:

- количество **FN** уменьшается  
- **Recall** растёт  
- но может увеличиваться число **FP**

Для медицинской задачи это допустимо, потому что **пропустить рак опаснее**, чем лишний раз заподозрить его.
