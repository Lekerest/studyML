## 📚 Датасет

Использован датасет **California Housing** (`fetch_california_housing` из `sklearn.datasets`).

Целевая переменная — **медианная стоимость дома** (в сотнях тысяч долларов).  
В данных присутствуют следующие признаки:

- `MedInc` — медианный доход в блоке  
- `HouseAge` — медианный возраст дома  
- `AveRooms` — среднее число комнат  
- `AveBedrms` — среднее число спален  
- `Population` — население блока  
- `AveOccup` — среднее число жильцов  
- `Latitude` — широта  
- `Longitude` — долгота  

---

## 🛠 Предобработка данных

### 1. Удаление цензурированных значений
В данных есть записи с максимальным значением `5.00001` (965 строк).  
Это искусственная обрезка — дома дороже 500 000 заменены на 500 001.  
Такие строки удалены, чтобы модель не училась на «обрезанных» данных.

### 2. Логарифмирование целевой переменной
Распределение цен сильно скошено вправо.  
Логарифмирование (`np.log`) сделало распределение более симметричным,  
что улучшает сходимость градиентного спуска.

### 3. Удаление признаков
- `AveBedrms` — сильно коррелирует с `AveRooms`, оставлен только `AveRooms`.  
- `Longitude` — исключён по рекомендации.

### 4. Добавление квадратичных признаков
Для учёта возможных нелинейных зависимостей добавлены квадраты:

- `HouseAge_sq` = `HouseAge`²  
- `Population_sq` = `Population`²  
- `MedInc_sq` = `MedInc`²  

Линейные исходные признаки сохранены — регуляризация сама определит их важность.

---

## 🏗 Архитектура модели

Модель представляет собой **один линейный слой** (`nn.Linear`):

\[
\hat{y} = X \cdot W + b
\]

### Параметры обучения

- Размер входного слоя: **9** (после добавления квадратов)
- Функция потерь: **MSE** (среднеквадратичная ошибка)
- Оптимизатор: **Adam** (`lr = 0.01`)
- L2‑регуляризация: **weight_decay = 0.01**
- Количество эпох: **500**
- Размер батча: **весь обучающий набор** (Batch Gradient Descent)

# ❗ Анализ мультиколлинеарности

Корреляция между `MedInc` и `MedInc_sq` = 0.96, между `HouseAge` и `HouseAge_sq` = 0.97.  
Это ожидаемо, но может привести к неустойчивости коэффициентов.  
Применённая L2‑регуляризация (`weight_decay`) сглаживает эту проблему,  
«сжимая» веса коррелированных признаков и снижая их дисперсию.  
Корреляция признаков остаётся высокой, но регуляризация предотвращает «перекос» весов, делая модель более устойчивой.
